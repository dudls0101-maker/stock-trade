# Windows 작업 스케줄러로 자동 실행하기

매일 미국 장 마감 30분 전쯤 `live_trader.py --live`가 자동으로 실행되도록 설정합니다.

## 1. 실행 배치 파일 만들기

먼저 `auto_trader/` 폴더에 `run_live.bat` 파일을 만드세요. 내용:

```
@echo off
cd /d C:\Users\indong\CascadeProjects\auto_trader
py -3.12 live_trader.py --live >> logs\scheduler.log 2>&1
```

저장 후 더블클릭해서 한 번 실행해보고 정상 동작하는지 확인.

## 2. 작업 스케줄러 등록

1. 시작 메뉴에서 "작업 스케줄러" 검색 → 실행
2. 우측 메뉴 "기본 작업 만들기"
3. 이름: `Alpaca Auto Trader`
4. 트리거: **매일**
5. 시작 시간:
   - 서머타임 적용 기간 (3월 둘째 일요일 ~ 11월 첫째 일요일): **새벽 4:30** (한국시간)
   - 표준시 기간: **새벽 5:30** (한국시간)
   - 시그널 계산은 일봉 기준이라 정확한 시간은 크게 중요하지 않음. 장 마감 1시간 전 ~ 30분 전 사이면 됨.
6. 동작: **프로그램 시작**
7. 프로그램/스크립트: `C:\Users\indong\CascadeProjects\auto_trader\run_live.bat`
8. **마침**

## 3. 주말 실행 방지 (선택)

토/일에는 미국 장이 안 열리니 실행 자체가 무의미. live_trader.py가 자동으로 "장 마감 중" 메시지만 출력하고 끝나기 때문에 그냥 둬도 손해는 없음.

## 4. 로그 확인

매 실행 후 다음 위치에 기록됩니다:

- `logs/scheduler.log` — 스케줄러로 실행된 모든 출력
- `logs/run_YYYYMMDD.log` — 오늘 실행한 상세 로그
- `logs/trades.csv` — 매수/매도 내역 누적
- Alpaca 대시보드 → Activity → 모든 주문 내역 확인 가능

## 5. 일시 중단

작업 스케줄러에서 해당 작업 우클릭 → "사용 안 함" 하면 다음부터 실행 안 됨.
다시 켜려면 "사용".

## 6. 첫 실행 권장 순서

1. **먼저 dry-run으로 며칠 돌려보기**: `py -3.12 live_trader.py` (--live 없이)
   - 시그널이 어떻게 계산되는지, 어떤 종목이 진입/청산 권고되는지 확인
2. **시그널이 합리적이면 --live로 한 번 수동 실행**: 첫 주문이 정상 들어가는지 Alpaca 대시보드에서 확인
3. **그 후 작업 스케줄러 등록**: 자동 실행
